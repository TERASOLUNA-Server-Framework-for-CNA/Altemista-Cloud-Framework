[[altemista-cloudfwk-app-oauth2-client-conf]]
=== OAuth 2.0 Client

In Java scenarios, the preferred way to implement an OAuth 2.0 client is using `OAuth2RestTemplate`, an implementation of Spring `RestTemplate` capable of making OAuth2-authenticated REST requests. Any Java client (and, in particular, a {framework} business-tier service implementation), can invoke an OAuth 2.0 Resource Server through an `OAuth2RestTemplate` object as follows:

.com.mycompany.application.module.service.impl.ExampleServiceImpl.java
[source]
----
@Autowired
private OAuth2RestTemplate restTemplate; // <1>

ResponseEntity<String> response = restOperations.getForEntity(url, String.class); //<2>
----
<1> Inject the declared `OAuth2RestTemplate` object. The declaration can be of any of the following types: `RestOperations`, `OAuth2RestOperations`, `RestTemplate` or `OAuth2RestTemplate`.
<2> Use as any other `RestTemplate` object.

According the decision chart for the different <<altemista-cloudfwk-security-auth2srv-conf-oauth2-overview-grants,OAuth 2.0 grant types>>, each grant type covers a different scenario; hence each grant type client is addressed to a different situation. Hence, {framework} only provides native ways of declaring the `OAuth2RestTemplate` instance for _Authorization Code Grant_ and _Client Credentials Grant_ flows.

==== Authorization Code Grant

The type of authorization Authorization Code Grant is used as soon as the client is a web server, and most likely any {framework} application invoking an OAuth 2.0 Resource Server will use this grant type.
