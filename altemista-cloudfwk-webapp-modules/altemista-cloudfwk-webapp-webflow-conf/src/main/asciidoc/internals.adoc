The module configures internally some beans that allows the detection of flows, handlers and the configuration need
to use this module.

[source,xml,options="nowrap"]
.org/altemista/cloudfwk/config/webapp/altemista-cloudfwk-webapp-webflow.xml
----
include::{basedir}/altemista-cloudfwk-webapp-webflow-conf/src/main/resources/org/altemista/cloudfwk/config/webapp/altemista-cloudfwk-webapp-webflow.xml[tags=internals]
----
<1> As previously stated, this default configuration looks for flow definitions at the locations `WEB-INF/flows/{asterisk}{asterisk}/{asterisk}-flow.xml` and `META-INF/flows/{asterisk}{asterisk}/{asterisk}-flow.xml`.
<2> A <<altemista-cloudfwk-webapp-conf-viewresolver,default `viewResolver`>> is included in the _webapp_ nature. Installing the Apache Tiles or JavaServer Faces (JSF) presentation layer features will supersede this `viewResolver`.

TIP: This configuration is compatible with the view technologies provided by the Apache Tiles and JavaServer Faces (JSF) features.
